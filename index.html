📜 The 8 BAT Scripts

Each script is purpose-built to solve a specific Git pain point

git-init.bat

Initialize a new Git repository and connect to GitHub

Setup

What It Does

One-command setup for new projects. Initializes Git, creates the main branch, and connects to your GitHub repository.

Usage

git-init.bat https://github.com/username/repo.git

Benefits

  • Single command to set up entire repo
  • Handles "remote already exists" error automatically
  • Forces main branch (modern standard)
  • Verifies connection at the end

git-push.bat

Stage, commit, and push all changes to GitHub

Daily Use

What It Does

The workhorse script you'll use daily. Prompts for a commit message, stages all files, commits, and pushes. If push fails, it offers a safe force-push option.

Usage

git-push.bat

Then type your commit message when prompted.

Benefits

  • Auto-defaults to date/time if no message provided
  • Falls back to force-push with confirmation
  • Window stays open so you see results
  • No need to remember Git syntax

git-pull.bat

Download latest from GitHub and overwrite local files

Destructive

What It Does

Fetches the latest code from GitHub and resets your local folder to match exactly. Deletes any uncommitted local changes.

Usage

git-pull.bat

Benefits

  • Perfect for multi-device workflows
  • No merge conflicts ever
  • Shows last 5 commits after sync
  • Cleans untracked files automatically

Warning

⚠️ All uncommitted local changes will be permanently deleted. Commit first if you want to keep them.

git-sync.bat

Smart sync — choose to pull, push, or do both

Smart

What It Does

Interactive menu that lets you choose: pull from GitHub, push to GitHub, or sync both ways. Perfect when you're not sure which direction you need.

Usage

git-sync.bat

Benefits

  • Interactive menu for flexibility
  • Shows status before doing anything
  • Handles all sync scenarios in one script
  • Great for beginners

git-status.bat

View branch, remote, commits, and current state

Info

What It Does

Quick overview of your Git state — current branch, remote URL, recent commits, and any uncommitted changes.

Usage

git-status.bat

Benefits

  • All info in one screen
  • No need to remember multiple Git commands
  • Perfect for debugging sync issues
  • Safe to run anytime

git-undo.bat

Undo the last commit (keep or delete changes)

Destructive

What It Does

Three options to undo commits: keep changes staged, unstage them, or delete everything. Hard reset requires typing "yes" to confirm.

Usage

git-undo.bat

Choose 1, 2, or 3 when prompted.

Benefits

  • Three levels of undo (safe to destructive)
  • Clear prompts explain what each does
  • Confirmation required for hard reset
  • Shows status after operation

git-branch.bat

Manage branches: create, switch, rename, delete

Manager

What It Does

Interactive branch management. Lists all branches, then lets you create, switch to, rename, or delete them with confirmations.

Usage

git-branch.bat

Benefits

  • All branch operations in one place
  • Safe deletion with confirmation
  • Shows local AND remote branches
  • No need to remember Git branch syntax

git-secret-check.bat

Scan staged files for API keys and secrets

Security

What It Does

Pre-commit security scanner. Checks your staged changes for common secret patterns (API keys, tokens, passwords) and warns you before pushing them to GitHub.

Usage

git-secret-check.bat

Benefits

  • Prevents accidental secret leaks
  • Catches common API key patterns
  • Saves you from GitHub push rejections
  • Run before any commit

Detects

  • API keys (sk-, pk-, api_key=)
  • Bearer tokens
  • Password assignments
  • Private keys (PEM format)

❓ FAQ

Common questions answered

Why use only the main branch?

Using a single branch eliminates 99% of merge conflicts. For solo projects or small teams, this is the fastest workflow. You can always add branches later when needed.

Is force-push safe?

For solo projects on the main branch: yes. For team projects: only force-push your own feature branches, never the main branch. Our scripts always ask for confirmation before force-pushing.

What if I lose files after git-pull.bat?

Run git reflog — it shows every commit you've been on, including "deleted" ones. Find your work, then git reset --hard <hash> to restore.

Can I use these on Mac/Linux?

These are Windows .bat files. For Mac/Linux, convert them to .sh bash scripts. The Git commands are the same — just change the syntax (e.g., set /p → read).

What if I accidentally commit a secret?

First, revoke the secret immediately (GitHub already blocked your push). Then remove it from your files, commit the fix, and force-push. Use git-secret-check.bat going forward.
